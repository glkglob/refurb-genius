# Deal Copilot — Future Integration Surface

> Status: **planned, not implemented.** This folder is reserved so the
> upcoming Deal Copilot product can plug into Refurb Genius's shared core
> without a rewrite. Do not add UI or live logic here yet.

Deal Copilot will be an acquisition-side product layered on top of the
existing Refurb Genius core. It reuses the same engines so a project
flowing through Deal Copilot ends up identical in shape to one created
manually inside Refurb Genius today.

## Reused shared core (do not duplicate)

| Concern              | Module                    | Notes                                                         |
| -------------------- | ------------------------- | ------------------------------------------------------------- |
| Project schema       | `@/core/projects`         | `Project`, `NewProjectInput`, `projectStore`, helpers.        |
| Refurb pricing       | `@/core/pricing`          | `runPricingEngine` — deterministic, region-aware.             |
| Investor metrics     | `@/core/roi`              | `runRoiEngine` — ROI, yield, profit, score, risk.             |
| Reporting            | `@/core/reports`          | `buildReport` — structured report object.                     |
| AI language / vision | `@/core/ai`               | Photo analysis, redesign concepts, summaries (no $ figures).  |

Money is generated by the pricing/ROI engines. AI never generates numbers.
Deal Copilot must follow the same boundary.

## Planned Deal Copilot surfaces

Each will live as its own submodule under `src/core/dealCopilot/` when built.
Files are intentionally absent today — they are listed here so the seams are
obvious to future contributors.

- `acquisitionPipeline/` — kanban-style stages (Sourced → Underwritten →
  Offer → Won/Lost). Wraps `projectStore` with a pipeline view; does not
  fork the project schema.
- `opportunities/` — candidate deals ingested from feeds (portals, off-market
  lists, agent emails). Each opportunity converts into a `Project` via
  `createProject` once the user chooses to underwrite it.
- `savedSearches/` — persisted filter sets (region, price band, yield
  threshold). Reuses `UK_REGIONS` and `PropertyType` from `@/core/projects`.
- `alerts/` — fan-out notifications when a new opportunity matches a saved
  search or when an underwritten project crosses a metric threshold.
- `monitoring/` — recurring re-scoring of live opportunities against the
  ROI engine; surfaces drift (price drops, yield improvements).
- `automation/` — rules engine that chains the above (e.g. "auto-create
  project when score ≥ 8 and region = London"). Always calls the existing
  engines; never re-implements pricing or ROI.

## Integration rules

1. **No schema fork.** Deal Copilot persists everything as `Project` rows
   plus sibling tables keyed by `project_id`. No parallel "deal" entity.
2. **No new pricing or ROI math.** All numeric output flows through
   `runPricingEngine` / `runRoiEngine`. New inputs (e.g. asking price vs.
   purchase price) are derived before calling the engines.
3. **AI stays language-only.** Deal Copilot's AI features (deal narratives,
   email summarisation, listing parsing) extend `@/core/ai` and must not
   produce financial figures.
4. **Reports are shared.** Acquisition reports extend `buildReport` with
   additional `ReportSection`s — never a parallel report builder.
5. **UI stays in `src/routes/deal-copilot/*`** when introduced. Do not edit
   existing Refurb Genius routes to add Deal Copilot behaviour; compose at
   the route layer instead.
