# Refurb IQ — Future Integration Surface

> Status: **planned, not implemented.** This folder is reserved so the
> upcoming Refurb IQ product can plug into Refurb Genius's shared core
> without a rewrite. Do not add UI or live logic here yet.

Refurb IQ will be a professional-side product (architects, surveyors, QSs,
contractors, project managers) layered on top of the same core that powers
Refurb Genius. It turns the homeowner/investor estimate into a contractor-
grade specification and cost plan.

## Reused shared core (do not duplicate)

| Concern              | Module                    | Notes                                                          |
| -------------------- | ------------------------- | -------------------------------------------------------------- |
| Project schema       | `@/core/projects`         | `Project`, `NewProjectInput`, `projectStore`, helpers.         |
| Estimate items       | `@/core/pricing`          | `PricingEstimateItem` + `runPricingEngine` line items.         |
| Refurb pricing       | `@/core/pricing`          | Region / condition / finish multipliers, VAT, contingency.     |
| Reporting            | `@/core/reports`          | `buildReport` + `ReportSection` for client-facing exports.     |
| AI language / vision | `@/core/ai`               | Photo analysis, room summaries, recommended works wording.     |

Money is generated by the pricing engine. AI never generates numbers.
Refurb IQ must follow the same boundary.

## Planned Refurb IQ surfaces

Each will live as its own submodule under `src/core/refurbIq/` when built.
Files are intentionally absent today — they are listed here so the seams are
obvious to future contributors.

- `boq/` — Bill of Quantities generation. Expands each
  `PricingEstimateItem` into NRM2 / SMM7 style measured items
  (description, quantity, unit, rate, total). Quantities derive from
  project size + room analysis; rates come from the pricing engine.
- `specification/` — Specification writer. Produces written work-section
  specs (preliminaries, demolition, M&E, finishes) using `@/core/ai`
  language helpers. AI writes prose; quantities and rates stay deterministic.
- `costPlans/` — Multi-stage cost plans (Stage 1 feasibility → Stage 4
  technical design). Wraps `runPricingEngine` with stage-level granularity
  and tracks variance against the original estimate.
- `contractors/` — Contractor workflow: tender packs, bid comparison,
  contractor pricing vs. engine baseline, valuation certificates, snagging.
  Reuses `Project` rows and `PricingEstimateItem` shape — no parallel job
  entity.
- `clientExports/` — Branded PDF/DOCX exports of BOQs, specs, cost plans,
  and valuations. Built on top of `buildReport` with extra `ReportSection`s
  and a Refurb IQ branding profile.

## Integration rules

1. **No schema fork.** Refurb IQ persists everything as `Project` rows plus
   sibling tables keyed by `project_id`. No parallel "job" or "tender" entity.
2. **No new pricing math.** Every rate / total flows through
   `runPricingEngine`. BOQs and cost plans expand the same line items;
   they never invent rates.
3. **AI stays language-only.** Specification writer, scope-of-works prose,
   and snag descriptions extend `@/core/ai`. AI must not produce
   quantities, rates, or totals.
4. **Reports are shared.** Refurb IQ exports extend `buildReport` with
   additional `ReportSection`s (BOQ table, spec sections, cost plan
   summary) — never a parallel report builder.
5. **UI stays in `src/routes/refurb-iq/*`** when introduced. Do not edit
   existing Refurb Genius routes to add Refurb IQ behaviour; compose at
   the route layer instead.
