# Legacy Dependency Audit - 2026-06-10

## Executive Summary

This audit identifies violations of the architectural boundary where code outside `src/features/` and `src/platform/` directly imports from legacy modules (`@/core`, `@/lib`, `@/services`, `@/integrations`).

## Audit Methodology

```bash
# Find all imports of legacy modules
grep -r --include="*.ts" --include="*.tsx" 'from ["'\'']@/(core|lib|services|integrations)' src/

# Specific legacy check
grep -r --include="*.ts" --include="*.tsx" 'core/ai/index\|lib/estimate\|integrations/supabase/client' src/
```

## Remediation Recommendations

### Immediate Actions (High Priority)

1. **Routes Layer** - Create facade/adapter in features:
   - Move `@/lib/analytics` → `@/features/analytics` or `@/platform/analytics`
   - Move `@/lib/logger` → `@/platform/logger`
   - Move `@/lib/sentry` → `@/platform/sentry`
   - Move `@/lib/utils` → `@/platform/utils`
   - Move `@/lib/auth` → `@/platform/auth`
   - Move `@/lib/queries/*` → corresponding feature repositories
   - Refactor routes to import from features/platform only

2. **Server Functions** - Align with feature boundaries:
   - `serverFns/projects.ts` → use `@/features/projects`
   - `serverFns/auth.ts` → use `@/platform/auth`

3. **UI Components** - Extract utility to platform:
   - Move `@/lib/utils` → `@/platform/utils/cn`
   - Update all `src/components/ui/*` imports

### Medium-Term Actions

4. **Component Decoupling**:
   - `AIMetricsDashboard.tsx` → create `@/features/ai-metrics` or move to platform

### Long-Term Actions

5. **Services Migration**:
   - Gradually migrate `src/services/trades/*` → `@/features/trades`
   - Migrate `src/services/projects/*` → `@/features/projects`
   - Migrate `src/services/storage/*` → `@/platform/storage`
